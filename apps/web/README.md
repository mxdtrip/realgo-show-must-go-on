# Web

Веб-приложение realgo на Next.js, React и TypeScript.

Приложение состоит из маркетингового лендинга и личного кабинета. Оформление построено на
едином `app/globals.css` без Tailwind/shadcn/ui — осознанное решение в пользу единой
GitHub Primer dark эстетики без слоя утилитарных классов.

## Маршруты

- `/` — маркетинговый лендинг.
- `/dashboard` — обзор подготовки (живые метрики, activity-хитмап, next action).
- `/reviews` — журнал повторений: очередь + история решённого с AI-подсказками (`assistant/hint`)
  и self-rating.
- `/problems` — личная база задач со стадиями практики.
- `/roadmap` — персональный план: недели считаются из даты собеседования, темы —
  из релевантных выбранной компании субпаттернов Pattern Atlas.
- `/patterns` — Pattern Atlas: 22 семейства / 111 субпаттернов, слабые паттерны, Company Overlay
  (что спрашивают конкретные компании) и лаунчер тренировки субпаттернов (`/me/practice`).
- `/cards` — лаунчер сессии повторения карточек (AI-генерация + сид-карточки).
- `/cards/session` — полноэкранная сессия повторения без элементов кабинета.
- `(focus)/cards`, `(focus)/patterns` — те же сессии в полноэкранном layout без кабинета.
- `/extension` — статус расширения.
- `/settings` — настройки аккаунта и приватности.
- `(marketing)/onboarding` — онбординг подготовки (площадка, компания, дата собеседования)
  с автоматическим предпросмотром персонального roadmap на последнем шаге.

Личный кабинет работает на живом backend API: auth, dashboard, roadmap, reviews, problems,
patterns/atlas, cards и extension-статус читают реальные данные. Единственный оставшийся мок —
`app/(cabinet)/cards/_mock.ts` (демо-данные лендингового flip-card превью), не используется
внутри самого кабинета.

## PWA и уведомления

- `app/manifest.ts` описывает PWA-приложение.
- `public/sw.js` регистрируется через `app/_pwa/PWAProvider.tsx`.
- `/settings` содержит панель установки приложения и настройки уведомлений.
- Флаги уведомлений синхронизируются с backend; browser permission, время напоминания и
  тестовое уведомление работают локально через service worker.

## Локализация

Все тексты вынесены в `app/_content/i18n.ts` — инфраструктура готова под 3 языка (`ru`,
`en`, `es`), компоненты читают строки по ключу, а не хардкодят текст. Основной язык
контента — русский; `en`/`es` подключаются без переписывания компонентов, когда понадобится.
